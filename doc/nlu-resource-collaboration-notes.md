# nlu+resource合作方式资源播放状态上报接口

## 背景

资源方对回传他们资源的播放状态要求很强硬，（包括用户ID（加密后）、声音音频ID、声音播放时长等等）——若违反则资源方将停止与百度的合作协议。

所以：`基于nlu+resource的合作方式，合作方也需要上报资源播放状态。`

ps:上报的端上的播放状态也为多轮解析提供了支撑。

声明：`合作方若没有按照规范上报客户端的播放状态，则需要自行承担相关资源方停止合作的风险`
